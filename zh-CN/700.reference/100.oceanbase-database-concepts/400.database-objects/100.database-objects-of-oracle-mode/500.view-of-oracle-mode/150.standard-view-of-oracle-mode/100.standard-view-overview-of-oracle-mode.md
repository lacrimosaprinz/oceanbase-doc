# 视图

## 概述

本章所讲的视图，是数据库中常用到的普通的视图，它是一种虚拟表的数据库对象，它提供了基于一个或多个实际表（即基表，基表可以是表或其他视图）的 SQL 查询结果的逻辑表示。视图不存储数据本身，而是存储了用于生成数据的查询。您可以通过访问这个视图来获取该视图定义的数据。

## 视图的优势

在数据库设计和应用中，视图提供了多方面的优势：

### 查询简化

* 把经常使用的数据定义为视图以简化操作。数据库的查询大多要使用聚合函数，同时还要显示其它字段的信息，可能还会需要关联到其它表，这时涉及的 SQL 语句可能比较复杂。如果需要频繁执行此查询，就可以通过创建视图简化查询操作，之后只需要执行 `SELECT * FROM view_name` 就可以获得预期结果。

### 数据安全性和隔离

* 视图限制用户只能查询和修改可视的数据，提高了数据的安全性。视图是动态的数据集合，数据随着基表的更新而更新。但是视图是虚拟的，物理上是不存在的，只是存储了数据的集合，所以可以将基表中重要的字段信息屏蔽，不通过视图展示给用户。

### 逻辑独立性

* 视图拥有逻辑上的独立性，屏蔽了真实表的结构带来的影响。视图可以使应用程序和数据库的表在一定程度上互相独立。如果没有视图，应用一定是建立在表上的。创建了视图之后，程序可以建立在视图之上，无需关心真实表结构的变化。

## 视图的特点

与实际的数据表不同，视图本身不占用存储空间用于数据存储。视图的特点包括：

- **存储的是查询**

  视图只存储定义它们的 SQL 查询语句，而非数据本身。

- **实时数据**

  由于视图是基于查询的，因此它们呈现的数据总是基表中最新的状态。

- **不额外占用存储空间**

  视图不需要为存放数据分配存储空间，除了在数据字典中存储其定义。

在使用表的大多数场合，可以用视图来代替实际的表，特别是在需要数据展示定制化、安全性和逻辑独立性方面的场景中。通过视图，数据库管理员和开发人员能够更有效地管理数据库对象，并为用户提供必要的数据访问服务。
