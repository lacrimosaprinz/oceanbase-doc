# connection_control_failed_connections_threshold

`connection_control_failed_connections_threshold` 用来指定用户错误登录尝试的阀值。

|        属性        |        描述        |
|------------------|------------------|
| 参数类型             | 整型               |
| 默认值              | 0                |
| 取值范围             | \[0，2147483647\] |
| 是否重启 OBServer 节点生效 | 否                |

将参数设置为 `0` 时，表示关闭该功能，即不限制用户的错误登录次数。当用户错误登录次数超过这个配置项定义的值之后，会对账户进行锁定，锁定的时长由 [connection_control_min_connection_delay](../400.tenant-level-configuration-items/600.connection_control_min_connection_delay.md) 和 [connection_control_max_connection_delay](../400.tenant-level-configuration-items/700.connection_control_max_connection_delay.md) 参数设置。

  <main id="notice" type='explain'>
    <h4>说明</h4>
    <p>该配置项当前仅在 MySQL 模式下生效。</p>
  </main>
