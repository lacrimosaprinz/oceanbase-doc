# 系统变量总览

## Global 系统变量

### 组复制相关

| 变量名                                                   | 功能描述                                                      |
|---------------------------------------------------------|---------------------------------------------------------------|
| group_replication_allow_local_disjoint_gtids_join       | 当服务器中具有组中不存在的本地事务时，是否允许其加入该组。其默认值为 `0`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>    |

| group_replication_auto_increment_increment              | 自动设置复制组中的每个成员的自增列的步长值，以确保在多主模式的组中，每个组成员的自增列值有序且不重叠。其默认值为 `7`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>       |
| group_replication_bootstrap_group                       | 指定使用当前服务器来 Bootstrap 组，该变量只能在一个服务器上设置，并且只能在首次时在 Bootstrap 组或重新 Bootstrap 组时在其中一个服务器上设置。其默认值为 `0`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>    |
| group_replication_components_stop_timeout                | 组复制关闭时，等待每个组件的超时时间，单位为秒。其默认值为 `2`。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>    |
| group_replication_compression_threshold                  | 组成员之间发送消息的阈值。当超过该阈值后，将会对组成员之间发送的消息进行压缩。其默认值为 `0`，表示禁用压缩。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| group_replication_enforce_update_everywhere_checks       | 在任何时候启用或禁用多主更新的严格一致性检查。其默认值为 `0`，表示禁用检查。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>    |
| group_replication_exit_state_action                      | 设置当服务器实例因异常而脱离组时，组复制的行为方式。其默认值为 `READ_ONLY`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| group_replication_flow_control_applier_threshold         | 指定触发流量控制的应用程序队列中等待事务的个数。其默认值为 `25000`。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| group_replication_flow_control_certifier_threshold       | 指定触发流控制的验证者队列中的等待事务的个数。其默认值为 `25000`。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| group_replication_flow_control_mode                      | 指定流控制所使用的模式。其默认值为 `QUOTA`。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| group_replication_force_members                          | 指定成员的地址列表，格式为 `host1:port1,host2:port2`。该变量用于强制建立一个新的组。设置成功后，被排除的成员（该变量中未指定的成员）将不会接收到新的视图消息并且还会被阻塞（阻塞写入操作）。其默认值为空。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>      |
| group_replication_group_name                             | 指定当前服务器所属的组复制的组名。其默认值为空。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| group_replication_group_seeds                            | 指定种子成员的地址列表，格式为 `host1:port1,host2:port2`。其默认值为空。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>    |
| group_replication_gtid_assignment_block_size             | 指定为每个成员保留的连续 GTID 数。其默认值为 `1000000`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| group_replication_ip_whitelist                           | 指定允许哪些主机（IP 地址或主机名）连接到组，被允许访问的地址称为白名单地址。其默认值为 `AUTOMATIC`。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| group_replication_local_address                          | 组复制成员用于提供给组中其他成员连接访问的网络地址。其默认值为空。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>     |
| group_replication_member_weight                          | 为组成员指定一个百分比权重，该权重值用于在发生故障转移时影响成员被选为主节点的机会（仅对处于单主模式的组有意义）。当处于单主模式的组中，其主节点脱离组时，权重值越高的成员越优先被选中为主节点。其默认值为 `50`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>     |
| group_replication_poll_spin_loops                        | 设置组复制通信线程在等待传入更多的网络消息之前，需要等待通信引擎互斥锁(mutex)被释放的次数。其默认值为 `0`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| group_replication_recovery_complete_at                   | 设置在分布式恢复过程中，完成状态传输之后应用缓存事务的策略。默认值为 `TRANSACTIONS_APPLIED`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| group_replication_recovery_reconnect_interval            | 当一个新的服务器尝试加入到组，由于某些原因导致无法找到可用的 Donor 进行分布式恢复时，会尝试重新连接去寻找可用的 Donor，该变量用于设置重新连接的间隔时间，单位为秒。其默认值为 `60`。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |

| group_replication_recovery_ssl_ca                        | 指定用于分布式恢复连接的受信任 SSL 证书颁发机构列表的文件的路径。其默认值为空。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| group_replication_recovery_ssl_capath                     | 指定用于分布式恢复连接的受信任的 SSL 证书颁发机构颁发的证书的目录路径。其默认值为空。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |

| group_replication_recovery_ssl_crl                        | 指定包含证书吊销列表文件的文件名。其默认值为空。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
