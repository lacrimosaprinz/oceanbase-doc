# 系统变量总览

## Global 系统变量

### 系统日志相关

| 变量名                                                   | 功能描述                                                      |
|---------------------------------------------------------|---------------------------------------------------------------|
| binlog_cache_size                                       | 仅用于设置事务缓存的大小，单位为字节，默认值为 `32768`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| binlog_direct_non_transactional_updates                 | 启用该变量会导致对非事务性表的更新直接写入 binlog，而不是写入事务缓存，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| binlog_error_action                                     | 控制服务器遇到错误时的行为，默认值为 `ABORT_SERVER`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| binlog_group_commit_sync_delay                          | 设置 binlog 提交后等待多长时间，才会将 binlog 文件同步到磁盘上，单位为微秒，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| binlog_group_commit_sync_no_delay_count                 | 指定等待延迟提交的最大事务数，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| binlog_max_flush_queue_time                             | 该变量值对系统没有任何影响，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| binlog_order_commits                                    | 当在复制源服务器上启用该变量时，向存储引擎发出的事务提交指令将在单个线程上序列化。其默认值为 `1`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| binlog_stmt_cache_size                                  | 该变量用于指定 binlog 的缓存大小，以便容纳事务期间发出的非事务语句。其默认值为 `32768`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| binlog_transaction_dependency_history_size              | 指定保存在内存中并用于查找上次修改指定行的事务的行哈希数的上限。其默认值为 `25000`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| binlog_transaction_dependency_tracking                  | 指定依赖项信息的来源，源使用该信息确定哪些事务可由副本的多线程应用程序并行执行，默认值为 `COMMIT_ORDER`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| expire_logs_days                                        | 设置 binlog 文件保留的时间，单位为天，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| log_bin_trust_function_creators                         | 控制是否可以信任存储函数创建者不会创建导致不安全事件写入 binlog 的存储函数，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| log_bin_use_v1_row_events                               | 控制是否正在使用版本 2 的 binlog，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| log_builtin_as_identified_by_password                   | 该变量影响用户管理语句的 binlog，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| max_binlog_cache_size                                   | 指定 binlog 能够使用的最大缓存大小，单位为字节。当事务所需的字节数超过该变量所指定的值时，服务器将会生成多语句事务需要超过 `max_binlog_cache_size` 字节的存储错误。其默认值为 `18446744073709500416`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| max_binlog_size                                         | 用于控制 binlog 文件的大小，它定义了 binlog 文件的最大容量，单位为字节。如果对 binlog 的写入导致当前日志文件大小超过该变量指定的值时，服务器会自动关闭当前 binlog 文件，并创建一个新的 binlog 文件来继续记录后续的数据库操作。其默认值为 `1073741824`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| max_binlog_stmt_cache_size                              | 指定在事务内执行非事务语句所需要使用的缓存大小，单位为字节，默认值为 `18446744073709547520`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| sync_binlog                                             | 用于控制服务器将 binlog 同步到磁盘的频率，默认值为 `1`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |

### 副本服务器相关

| 变量名                                                   | 功能描述                                                      |
|---------------------------------------------------------|---------------------------------------------------------------|
| max_relay_log_size                                      | 指定服务器自动轮换中继日志文件的大小阈值，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| relay_log_info_repository                               | 指定副本服务器是将其应用元数据存储库作为 InnoDB 表保存在 mysql 系统数据库中，还是作为数据目录中的一个文件，默认值为 `FILE`。  <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| relay_log_purge                                         | 对不再需要的中继日志文件，启用或禁用自动清除功能，默认值为 `1`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>   |
| sync_relay_log                                          | 设置如何同步中继日志到中继日志文件。当该变量的值大于 0 时，MySQL 服务器会在向中继日志写入每 `sync_relay_log` 次事件后，同步中继日志到磁盘。其默认值为 `10000`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| sync_relay_log_info                                     | 设置如何将应用中继日志的位置信息同步到文件或表中，设置此变量会立即对所有复制通道生效，包括正在运行的通道。其默认值为 `10000`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
