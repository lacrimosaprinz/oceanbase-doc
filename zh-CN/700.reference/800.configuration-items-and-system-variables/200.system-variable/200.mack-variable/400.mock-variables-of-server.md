# 系统变量总览

## Global 系统变量

### 系统服务器相关

| 变量名                                            | 功能描述                                                      |
|---------------------------------------------------|---------------------------------------------------------------|
| join_buffer_size                                  | 指定用于纯索引扫描、范围索引扫描以及不使用索引而执行全表扫描的联接的缓冲区的最小大小，单位为字节，默认值为 `262144`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| max_join_size                                     | 用于限制 Join 操作中允许的最大行数，默认值为 `18446744073709547520`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| max_length_for_sort_data                          | 指定索引值大小的截断值，用于确定使用哪种文件排序算法，默认值为 `1024`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| max_prepared_stmt_count                           | 用于限制服务器中预处理语句的条数，默认值为 `16382`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| max_sort_length                                   | 对数值进行排序时所使用的字节数，默认值为 `1024`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>|
| min_examined_row_limit                            | 指定检查的行数大于等于多少行时会记录到慢查询日志中，即检查少于该变量指定的行数的查询不会记录到慢查询日志中，默认值为 `0`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| multi_range_count                                 | 指定查询语句中 Range 范围的最大个数，默认值为 `256`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| query_alloc_block_size                            | 为语句解析和执行期间创建的对象所分配的内存块的大小，单位为字节，默认值为 `8192`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| query_prealloc_size                               | 用于语句解析和执行的固定缓冲区的大小，单位为字节，默认值为 `8192`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| slow_query_log                                    | 控制是否启用慢查询日志，默认值为 `0`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| slow_query_log_file                               | 指定慢查询日志的文件名，默认值为 `/usr/local/mysql/data/obrd-slow.log`。<main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| sort_buffer_size                                  | 指定执行排序时使用的缓冲区大小，单位为字节，默认值为 `262144`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |
| sql_buffer_result                                  | 控制是否将 `SELECT` 语句的结果放入临时表中，默认值为 `0`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main>  |


### X Plugin 相关

| 变量名                                            | 功能描述                                                      |
|---------------------------------------------------|---------------------------------------------------------------|
| mysqlx_connect_timeout                            | X 插件等待从新连接的客户端接收第一个数据包的时间，单位为秒，默认值为 `30`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| mysqlx_idle_worker_thread_timeout                 | 指定空闲工作线程在被终止之前可以保持空闲状态的时间，单位为秒，默认值为 `60`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| mysqlx_max_allowed_packet                         | X 插件可以接收的网络数据包的最大大小，单位为字节，默认值为 `67108864`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| mysqlx_max_connections                            | X 插件可以接受的最大并发客户端连接数，默认值为 `100`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
| mysqlx_min_worker_threads                         | X 插件用于处理客户端请求的最小工作线程数，默认值为 `2`。 <main id="notice" type='explain'><h4>说明</h4><p>该变量仅用于兼容 MySQL 5.7，相关功能并不支持，可以查询及设置，设置行为不报错但不生效，会产生 Warning。</p></main> |
