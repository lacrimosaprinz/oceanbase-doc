# resource_manager_plan

`resource_manager_plan` 用于激活资源管理计划。指定不同计划，会使用不同 CPU 的额度来限定资源组下的用户活动。

|   属性   |      描述       |
|--------|---------------|
| 参数类型   | varchar       |
| 默认值    | ""            |
| 取值范围   | N/A           |
| 生效范围   | GLOBAL        |
| 是否重启生效 | 否，设置后 10s 内生效 |

租户内的普通线程和并行执行线程会相互抢占 CPU 资源，通过为租户指定资源隔离计划，可以控制资源抢占策略，限制资源的使用量。
